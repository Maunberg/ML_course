Сверточная сеть (CNN)

CNN для классификации изображений

Принцип работы коротко

Свертки извлекают локальные паттерны, pooling уменьшает размерность, полносвязная голова классифицирует.

Математическое обоснование
2D convolution:
y[n, c_out, i, j] = sum_{c_in, u, v} W[c_out, c_in, u, v] * x[n, c_in, i+u, j+v]

Далее:
h = ReLU(BN(y))
h_pool = MaxPool(h)
logits = Linear(flatten(h_pool))

Loss:
L = CrossEntropy(logits, y_true)

Обучение:
backprop через conv, bn, linear + шаг Adam/SGD.
Реализация PyTorch
import torch
import torch.nn as nn

class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(3, 32, kernel_size=3, padding=1),
            nn.BatchNorm2d(32),
            nn.ReLU(),
            nn.MaxPool2d(2),
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(),
            nn.MaxPool2d(2)
        )
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(64 * 8 * 8, 128),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(128, num_classes)
        )

    def forward(self, x):
        x = self.features(x)
        return self.classifier(x)

model = SimpleCNN(num_classes=10)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
Описание параметров реализации PyTorch
  • out_channels в Conv2d: сколько карт признаков извлекаем.
  • kernel_size: размер окна свертки (часто 3x3).
  • stride, padding: контроль пространственного размера.
  • BatchNorm2d: стабилизация и ускорение обучения.
  • Dropout: регуляризация.
  • num_classes: размер выходного слоя.
  • lr, weight_decay, batch_size: ключевые параметры оптимизации.
Интуиция использования

Стандартный выбор для изображений и карт признаков, когда важны локальные шаблоны и их иерархия.
