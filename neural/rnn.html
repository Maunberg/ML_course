LSTM для последовательностей

Принцип работы коротко

Модель обрабатывает последовательность по шагам и хранит состояние, которое переносит информацию из прошлого.

Математическое обоснование
Для LSTM на шаге t:
f_t = sigmoid(W_f x_t + U_f h_{t-1} + b_f)      # forget gate
i_t = sigmoid(W_i x_t + U_i h_{t-1} + b_i)      # input gate
g_t = tanh(W_g x_t + U_g h_{t-1} + b_g)         # candidate
c_t = f_t * c_{t-1} + i_t * g_t                 # memory update
o_t = sigmoid(W_o x_t + U_o h_{t-1} + b_o)      # output gate
h_t = o_t * tanh(c_t)

Loss:
L = CrossEntropy(head(h_T), y_true)

Обучение:
BPTT (backpropagation through time) + optimizer step.
Реализация PyTorch
import torch
import torch.nn as nn

class LSTMClassifier(nn.Module):
    def __init__(self, input_size, hidden_size=128, num_layers=2, num_classes=2, dropout=0.2):
        super().__init__()
        self.lstm = nn.LSTM(
            input_size=input_size,
            hidden_size=hidden_size,
            num_layers=num_layers,
            batch_first=True,
            dropout=dropout if num_layers > 1 else 0.0,
            bidirectional=False
        )
        self.head = nn.Linear(hidden_size, num_classes)

    def forward(self, x):
        out, (h_n, c_n) = self.lstm(x)
        last_hidden = h_n[-1]
        logits = self.head(last_hidden)
        return logits

model = LSTMClassifier(input_size=64, hidden_size=128, num_layers=2, num_classes=3)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
Описание параметров реализации PyTorch
  • input_size: размер признаков на шаге последовательности.
  • hidden_size: размер скрытого состояния (емкость памяти).
  • num_layers: глубина стека LSTM.
  • dropout: применяется между LSTM-слоями.
  • bidirectional: двунаправленный контекст.
  • batch_first=True: формат входа (batch, seq, feature).
  • lr, clip_grad_norm_: стабильность обучения.
Интуиция использования

Подходит для временных рядов, текста и событийных последовательностей, где порядок наблюдений критичен.
