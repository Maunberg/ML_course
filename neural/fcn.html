Полносвязная сеть (FCN / MLP)

MLP для классификации

Принцип работы коротко

Вход проходит через последовательность линейных слоев и нелинейностей, на выходе получаем логиты классов.

Математическое обоснование
Forward pass:
h1 = ReLU(W1 x + b1)
h2 = ReLU(W2 h1 + b2)
logits = W3 h2 + b3
p(y=c|x) = softmax(logits)_c

Loss (cross-entropy):
L = - (1/N) * sum_i log p(y_i | x_i)

Обучение:
1) forward
2) вычисление loss
3) backward (autograd: dL/dW, dL/db)
4) шаг оптимизатора (SGD/Adam)
Реализация PyTorch
import torch
import torch.nn as nn

class MLPClassifier(nn.Module):
    def __init__(self, in_features, num_classes, hidden1=128, hidden2=64, dropout=0.2):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(in_features, hidden1),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(hidden1, hidden2),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(hidden2, num_classes)
        )

    def forward(self, x):
        return self.net(x)

model = MLPClassifier(in_features=100, num_classes=5)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-4)

for xb, yb in train_loader:
    optimizer.zero_grad()
    logits = model(xb)
    loss = criterion(logits, yb)
    loss.backward()
    optimizer.step()
Описание параметров реализации PyTorch
  • in_features: размер входного вектора.
  • num_classes: число классов.
  • hidden1, hidden2: ширина скрытых слоев (емкость модели).
  • dropout: регуляризация, снижает переобучение.
  • lr: шаг обучения.
  • weight_decay: L2-регуляризация в оптимизаторе.
  • batch_size: стабильность градиента и скорость.
  • epochs: число проходов по train.
Интуиция использования

Хороший выбор для табличных/векторных признаков, когда линейной модели уже не хватает, но архитектура CNN/Transformer избыточна.
