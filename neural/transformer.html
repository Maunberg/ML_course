Трансформер

Encoder Transformer для классификации

Принцип работы коротко

Self-attention позволяет каждому токену учитывать весь контекст последовательности на каждом слое.

Математическое обоснование
Scaled dot-product attention:
Q = X W_Q, K = X W_K, V = X W_V
Attention(Q, K, V) = softmax( (Q K^T) / sqrt(d_k) ) V

Multi-head:
head_i = Attention(Q_i, K_i, V_i)
MHA(X) = Concat(head_1, ..., head_h) W_O

Encoder block:
X1 = LayerNorm(X + MHA(X))
X2 = LayerNorm(X1 + FFN(X1))

Loss:
L = CrossEntropy(classifier(X_cls), y_true)
Реализация PyTorch
import torch
import torch.nn as nn

class TransformerClassifier(nn.Module):
    def __init__(self, vocab_size, d_model=128, nhead=4, num_layers=2, dim_ff=256, num_classes=2, max_len=512):
        super().__init__()
        self.token_emb = nn.Embedding(vocab_size, d_model)
        self.pos_emb = nn.Embedding(max_len, d_model)
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=d_model,
            nhead=nhead,
            dim_feedforward=dim_ff,
            dropout=0.1,
            batch_first=True
        )
        self.encoder = nn.TransformerEncoder(encoder_layer, num_layers=num_layers)
        self.head = nn.Linear(d_model, num_classes)

    def forward(self, x):
        bsz, seq_len = x.shape
        pos = torch.arange(seq_len, device=x.device).unsqueeze(0).expand(bsz, seq_len)
        h = self.token_emb(x) + self.pos_emb(pos)
        h = self.encoder(h)
        cls = h[:, 0, :]
        return self.head(cls)

model = TransformerClassifier(vocab_size=30000, d_model=256, nhead=8, num_layers=4, num_classes=3)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.AdamW(model.parameters(), lr=3e-4, weight_decay=1e-2)
Описание параметров реализации PyTorch
  • vocab_size: размер словаря токенов.
  • d_model: размер эмбеддинга и внутреннего представления.
  • nhead: число attention-голов (должно делить d_model).
  • num_layers: глубина энкодера.
  • dim_feedforward: ширина FFN блока.
  • max_len: максимальная длина последовательности для positional embedding.
  • lr, weight_decay, scheduler/warmup: критичны для стабильного обучения.
Интуиция использования

Лучший выбор для задач с длинным контекстом и сложными зависимостями между токенами (NLP, код, мультимодальные последовательности).
