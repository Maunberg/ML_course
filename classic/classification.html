1. Naive Bayes

Коротко: как работает

Модель считает вероятность класса по формуле Байеса и выбирает класс с максимальной апостериорной вероятностью. Главная идея — признаки считаются условно независимыми внутри класса.

Когда полезен

Очень быстрый baseline. Особенно хорош для текста, разреженных матриц, простых задач классификации и небольших обучающих выборок.

Подробное математическое объяснение

P(y=c \mid x)=\frac{P(x \mid y=c)P(y=c)}{P(x)}

Так как P(x) одинаково для всех классов, при сравнении классов можно использовать:

P(y=c \mid x) \propto P(y=c)P(x \mid y=c)

Наивное предположение независимости:

P(x \mid y=c)=\prod_{j=1}^{d} P(x_j \mid y=c)

Тогда итоговый скор:

P(y=c \mid x) \propto P(y=c)\prod_{j=1}^{d} P(x_j \mid y=c)

Для численной устойчивости почти всегда переходят к логарифмам:

\log P(y=c \mid x) = \log P(y=c)+\sum_{j=1}^{d}\log P(x_j \mid y=c)+const

Gaussian Naive Bayes: математика

Если признак непрерывный, часто предполагают нормальное распределение внутри класса:

P(x_j \mid y=c)=\frac{1}{\sqrt{2\pi\sigma_{cj}^2}}\exp\left(-\frac{(x_j-\mu_{cj})^2}{2\sigma_{cj}^2}\right)

Параметры оцениваются по обучающей выборке:

\mu_{cj}=\frac{1}{N_c}\sum_{i:y_i=c} x_{ij}, \quad \sigma^2_{cj}=\frac{1}{N_c}\sum_{i:y_i=c}(x_{ij}-\mu_{cj})^2
P(y=c)=\frac{N_c}{N}

Собственная реализация с объяснением

Ниже реализация Gaussian Naive Bayes. Что делает модель:

  • на этапе fit считает средние, дисперсии и априорные вероятности классов;
  • на этапе predict считает логарифм правдоподобия для каждого класса;
  • выбирает класс с максимальным значением.
import numpy as np

class GaussianNaiveBayes:
    def __init__(self, var_smoothing=1e-9):
        self.var_smoothing = var_smoothing
        self.classes_ = None
        self.means_ = None
        self.vars_ = None
        self.priors_ = None

    def fit(self, X, y):
        X = np.asarray(X, dtype=float)
        y = np.asarray(y)

        n_samples, n_features = X.shape
        self.classes_ = np.unique(y)
        n_classes = len(self.classes_)

        self.means_ = np.zeros((n_classes, n_features))
        self.vars_ = np.zeros((n_classes, n_features))
        self.priors_ = np.zeros(n_classes)

        for idx, c in enumerate(self.classes_):
            X_c = X[y == c]
            self.means_[idx] = X_c.mean(axis=0)
            self.vars_[idx] = X_c.var(axis=0) + self.var_smoothing
            self.priors_[idx] = X_c.shape[0] / n_samples

    def _log_likelihood(self, class_idx, x):
        mean = self.means_[class_idx]
        var = self.vars_[class_idx]
        prior = np.log(self.priors_[class_idx])

        log_prob = -0.5 * np.sum(np.log(2 * np.pi * var))
        log_prob -= 0.5 * np.sum(((x - mean) ** 2) / var)

        return prior + log_prob

    def predict(self, X):
        X = np.asarray(X, dtype=float)
        y_pred = []

        for x in X:
            posteriors = [self._log_likelihood(i, x) for i in range(len(self.classes_))]
            y_pred.append(self.classes_[np.argmax(posteriors)])

        return np.array(y_pred)

2. K-Nearest Neighbors (KNN)

Коротко: как работает

Для нового объекта ищем k ближайших соседей в обучающей выборке и выбираем класс по большинству голосов или по взвешенному голосованию.

Главная идея

Похожие объекты должны иметь похожие метки. KNN ничего не обучает в привычном смысле — он просто запоминает train-набор.

Подробное математическое объяснение

Для точки x ищется множество ближайших соседей N_k(x) по метрике расстояния.

\hat{y}=\arg\max_c \sum_{i \in N_k(x)} w_i \cdot I(y_i=c)

Если веса одинаковые, то w_i = 1. Если используется взвешивание по расстоянию:

w_i = \frac{1}{d(x, x_i) + \varepsilon}

Часто используется расстояние Минковского:

d(x, z)=\left(\sum_{j=1}^{d}|x_j-z_j|^p\right)^{1/p}
  • p=1 — Manhattan distance
  • p=2 — Euclidean distance

В KNN нет явной функции потерь и градиентного спуска: всё обучение — это сохранение выборки. Поэтому он очень прост, но предсказание может быть медленным.

Собственная реализация с объяснением

import numpy as np
from collections import Counter

class KNNClassifier:
    def __init__(self, n_neighbors=5, weights="uniform", p=2):
        self.n_neighbors = n_neighbors
        self.weights = weights
        self.p = p
        self.X_train = None
        self.y_train = None

    def fit(self, X, y):
        self.X_train = np.asarray(X, dtype=float)
        self.y_train = np.asarray(y)

    def _distance(self, x1, x2):
        return np.sum(np.abs(x1 - x2) ** self.p) ** (1 / self.p)

    def predict(self, X):
        X = np.asarray(X, dtype=float)
        predictions = []

        for x in X:
            distances = np.array([self._distance(x, x_train) for x_train in self.X_train])
            k_idx = np.argsort(distances)[:self.n_neighbors]
            k_labels = self.y_train[k_idx]

            if self.weights == "distance":
                k_distances = distances[k_idx]
                weights = 1 / (k_distances + 1e-12)
                scores = {}
                for label, w in zip(k_labels, weights):
                    scores[label] = scores.get(label, 0) + w
                pred = max(scores, key=scores.get)
            else:
                pred = Counter(k_labels).most_common(1)[0][0]

            predictions.append(pred)

        return np.array(predictions)

Что здесь происходит

  • fit просто сохраняет обучающие данные;
  • для каждого нового объекта считаются расстояния до всех объектов train;
  • выбираются k ближайших;
  • итоговый класс определяется голосованием.

3. Logistic Regression

Коротко: как работает

Логистическая регрессия строит линейную комбинацию признаков, потом пропускает её через сигмоиду и получает вероятность класса.

Интуиция

Модель пытается провести гиперплоскость, разделяющую классы, и сделать уверенные вероятностные предсказания.

Математическое объяснение

z = Xw + b
\sigma(z)=\frac{1}{1+e^{-z}}
\hat{y}=\sigma(Xw+b)

Функция потерь для бинарного случая:

L = -\frac{1}{n}\sum_{i=1}^{n}\left[y_i\log(\hat{y}_i)+(1-y_i)\log(1-\hat{y}_i)\right]

Градиенты:

\frac{\partial L}{\partial w}=\frac{1}{n}X^T(\hat{y}-y), \quad \frac{\partial L}{\partial b}=\frac{1}{n}\sum_{i=1}^{n}(\hat{y}_i-y_i)

Обновление параметров:

w := w - \eta \frac{\partial L}{\partial w}, \quad b := b - \eta \frac{\partial L}{\partial b}

Разделяющая граница:

Xw+b=0

Собственная реализация с объяснением

import numpy as np

class LogisticRegressionCustom:
    def __init__(self, lr=0.01, n_iters=1000):
        self.lr = lr
        self.n_iters = n_iters
        self.w = None
        self.b = None

    def _sigmoid(self, z):
        return 1 / (1 + np.exp(-z))

    def fit(self, X, y):
        X = np.asarray(X, dtype=float)
        y = np.asarray(y, dtype=float)
        n_samples, n_features = X.shape

        self.w = np.zeros(n_features)
        self.b = 0.0

        for _ in range(self.n_iters):
            linear_model = np.dot(X, self.w) + self.b
            y_pred = self._sigmoid(linear_model)

            dw = (1 / n_samples) * np.dot(X.T, (y_pred - y))
            db = (1 / n_samples) * np.sum(y_pred - y)

            self.w -= self.lr * dw
            self.b -= self.lr * db

    def predict_proba(self, X):
        X = np.asarray(X, dtype=float)
        linear_model = np.dot(X, self.w) + self.b
        return self._sigmoid(linear_model)

    def predict(self, X, threshold=0.5):
        probs = self.predict_proba(X)
        return (probs >= threshold).astype(int)

Логика обучения:

  • инициализируем веса нулями;
  • считаем линейную комбинацию признаков;
  • получаем вероятности через sigmoid;
  • считаем ошибку и градиенты;
  • делаем шаг градиентного спуска.

4. Decision Tree

Коротко: как работает

Decision Tree последовательно делит пространство признаков на области с помощью правил вида x_j \le t, пока листья не станут достаточно "чистыми".

Интуиция

Дерево задаёт цепочку вопросов: «признак меньше порога?». В конце пути попадаем в лист и получаем класс.

Подробное математическое объяснение

В каждом узле выбирается разбиение, которое максимально уменьшает неоднородность.

Для классификации обычно используются:

Gini = 1 - \sum_{c=1}^{K} p_c^2
Entropy = -\sum_{c=1}^{K} p_c \log p_c

Качество разбиения:

\Delta I = I(parent) - \left(\frac{n_{left}}{n}I(left) + \frac{n_{right}}{n}I(right)\right)

Здесь I — мера неоднородности. Алгоритм жадный: на каждом шаге выбирается лучший локальный split.

Собственная реализация с объяснением

import numpy as np

class DecisionTreeNode:
    def __init__(self, feature=None, threshold=None, left=None, right=None, value=None):
        self.feature = feature
        self.threshold = threshold
        self.left = left
        self.right = right
        self.value = value

class DecisionTreeClassifierCustom:
    def __init__(self, max_depth=10, min_samples_split=2):
        self.max_depth = max_depth
        self.min_samples_split = min_samples_split
        self.root = None

    def _gini(self, y):
        classes, counts = np.unique(y, return_counts=True)
        probs = counts / len(y)
        return 1 - np.sum(probs ** 2)

    def _best_split(self, X, y):
        n_samples, n_features = X.shape
        best_gain = -1
        split_idx, split_thresh = None, None
        parent_impurity = self._gini(y)

        for feature_idx in range(n_features):
            thresholds = np.unique(X[:, feature_idx])
            for threshold in thresholds:
                left_mask = X[:, feature_idx] <= threshold
                right_mask = ~left_mask

                if np.sum(left_mask) == 0 or np.sum(right_mask) == 0:
                    continue

                left_y = y[left_mask]
                right_y = y[right_mask]

                n_left, n_right = len(left_y), len(right_y)
                child_impurity = (n_left / n_samples) * self._gini(left_y) + \
                                 (n_right / n_samples) * self._gini(right_y)

                info_gain = parent_impurity - child_impurity

                if info_gain > best_gain:
                    best_gain = info_gain
                    split_idx = feature_idx
                    split_thresh = threshold

        return split_idx, split_thresh

    def _build_tree(self, X, y, depth=0):
        n_samples = X.shape[0]
        n_labels = len(np.unique(y))

        if depth >= self.max_depth or n_labels == 1 or n_samples < self.min_samples_split:
            leaf_value = np.bincount(y).argmax()
            return DecisionTreeNode(value=leaf_value)

        feature_idx, threshold = self._best_split(X, y)
        if feature_idx is None:
            leaf_value = np.bincount(y).argmax()
            return DecisionTreeNode(value=leaf_value)

        left_mask = X[:, feature_idx] <= threshold
        right_mask = ~left_mask

        left_child = self._build_tree(X[left_mask], y[left_mask], depth + 1)
        right_child = self._build_tree(X[right_mask], y[right_mask], depth + 1)

        return DecisionTreeNode(feature_idx, threshold, left_child, right_child)

    def fit(self, X, y):
        X = np.asarray(X, dtype=float)
        y = np.asarray(y, dtype=int)
        self.root = self._build_tree(X, y)

    def _traverse_tree(self, x, node):
        if node.value is not None:
            return node.value

        if x[node.feature] <= node.threshold:
            return self._traverse_tree(x, node.left)
        return self._traverse_tree(x, node.right)

    def predict(self, X):
        X = np.asarray(X, dtype=float)
        return np.array([self._traverse_tree(x, self.root) for x in X])

Что делает этот код

  • считает impurity в узле через Gini;
  • перебирает все признаки и пороги;
  • находит split с максимальным приростом качества;
  • рекурсивно строит левое и правое поддерево;
  • в листьях хранит наиболее частый класс.

Плюсы
Очень интерпретируемая модель, не требует масштабирования, хорошо ловит нелинейности и взаимодействия признаков.
Минусы
Легко переобучается, нестабильна к малым изменениям данных, иногда даёт рваные разделяющие границы.
Сколько данных нужно: работает от малых до средних выборок; на больших данных часто переходят к ансамблям типа Random Forest и Gradient Boosting.
Лучшие задачи: табличные данные, объяснимые модели, задачи с нелинейными зависимостями, когда важна интерпретация правил.
Важно: деревья не требуют нормализации признаков.

Модель Сильные стороны Слабые стороны Где особенно хороша
Naive Bayes Очень быстрый, хорош на тексте, мало данных для старта Независимость признаков часто нарушается Тексты, спам, baseline

Logistic Regression Интерпретируемая, быстрая, даёт вероятности Плохо для сложных нелинейностей Линейно разделимые табличные данные, baseline
