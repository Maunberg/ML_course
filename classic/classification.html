Naive Bayes (NB)

Принцип работы коротко

Считает вероятность каждого класса по формуле Байеса и выбирает класс с максимальной вероятностью, предполагая независимость признаков.

Математическое обоснование подробно

Используем P(y|x) ∝ P(y) Π P(x_i|y). Для непрерывных признаков часто берут гауссово распределение: P(x_i|y=c)=N(μ_{ci},σ_{ci}^2). На практике работаем в логарифмах: log P(y)+Σ log P(x_i|y).

Собственная реализация
class GaussianNBFromScratch:
    def fit(self, X, y):
        ...
    def predict(self, X):
        ...
Реализация sklearn
from sklearn.naive_bayes import GaussianNB
model = GaussianNB(var_smoothing=1e-9)
model.fit(X_train, y_train)
Описание гиперпараметров
  • var_smoothing - численная стабильность.
  • priors - априорные вероятности классов.
Интуиция использования (когда, какие данные, плюсы/минусы)
  • Когда: быстрый baseline, особенно текст и разреженные признаки.
  • Плюсы: скорость, простота, устойчивость на малых данных.
  • Минусы: сильное предположение независимости признаков.
Сетка гиперпараметров для гридсёрч
param_grid = {
    "var_smoothing": [1e-12, 1e-10, 1e-9, 1e-8, 1e-7]
}

K Nearest Neighbors (KNN)

Принцип работы коротко

Находит ближайшие объекты из train и голосует их классами. Для регрессии усредняет значения соседей.

Математическое обоснование подробно

Для точки x ищем N_k(x) по метрике d(x, x_i). Класс выбираем как argmax_c Σ I(y_i=c) по соседям. Взвешенный вариант использует веса w_i=1/(d_i+ε).

Собственная реализация
class KNNFromScratch:
    def __init__(self, k=5):
        self.k = k
    def fit(self, X, y):
        self.X, self.y = X, y
    def predict(self, X):
        ...
Реализация sklearn
from sklearn.neighbors import KNeighborsClassifier
model = KNeighborsClassifier(n_neighbors=5, weights="distance")
model.fit(X_train, y_train)
Описание гиперпараметров
  • n_neighbors - сколько соседей учитывать.
  • weights - равные веса или по расстоянию.
  • metric, p - метрика расстояния.
Интуиция использования (когда, какие данные, плюсы/минусы)
  • Когда: небольшие/средние датасеты, нелинейные границы.
  • Плюсы: не требует обучения параметров модели.
  • Минусы: медленный инференс, чувствителен к масштабу признаков.
Сетка гиперпараметров для гридсёрч
param_grid = {
    "n_neighbors": [3, 5, 7, 9, 11],
    "weights": ["uniform", "distance"],
    "metric": ["euclidean", "manhattan"],
    "p": [1, 2]
}

Random Forest (RF)

Принцип работы коротко

Ансамбль деревьев на бутстрэп-выборках со случайным выбором признаков. Итог - голосование деревьев.

Математическое обоснование подробно

Каждое дерево обучается на бутстрэпе D_b. На каждом узле выбирается лучший сплит из случайного подмножества признаков. Усреднение/голосование снижает дисперсию и переобучение.

Собственная реализация
class RandomForestToy:
    def __init__(self, n_estimators=100):
        self.n_estimators = n_estimators
    def fit(self, X, y):
        ...
    def predict(self, X):
        ...
Реализация sklearn
from sklearn.ensemble import RandomForestClassifier
model = RandomForestClassifier(n_estimators=300, random_state=42)
model.fit(X_train, y_train)
Описание гиперпараметров
  • n_estimators - число деревьев.
  • max_depth - ограничение глубины.
  • min_samples_split, min_samples_leaf - регуляризация.
  • max_features - число признаков на сплит.
Интуиция использования (когда, какие данные, плюсы/минусы)
  • Когда: табличные данные, нужен сильный baseline.
  • Плюсы: устойчивость, неплохая работа из коробки.
  • Минусы: крупная модель, ниже интерпретируемость.
Сетка гиперпараметров для гридсёрч
param_grid = {
    "n_estimators": [100, 200, 400],
    "max_depth": [None, 5, 10, 20],
    "min_samples_split": [2, 5, 10],
    "min_samples_leaf": [1, 2, 4],
    "max_features": ["sqrt", "log2"]
}

Decision Tree (DT)

Принцип работы коротко

Последовательно делит пространство признаков на области с максимально однородными классами.

Математическое обоснование подробно

На каждом узле ищем сплит с максимальным уменьшением неопределенности: ΔI = I(parent) - Σ (n_j/n) I(child_j), где I - Gini или Entropy.

Собственная реализация
class DecisionTreeOneSplit:
    def fit(self, X, y):
        ...
    def predict(self, X):
        ...
Реализация sklearn
from sklearn.tree import DecisionTreeClassifier
model = DecisionTreeClassifier(random_state=42)
model.fit(X_train, y_train)
Описание гиперпараметров
  • criterion - gini или entropy.
  • max_depth - глубина дерева.
  • min_samples_split, min_samples_leaf - контроль переобучения.
  • ccp_alpha - pruning.
Интуиция использования (когда, какие данные, плюсы/минусы)
  • Когда: нужна объяснимость и простые правила.
  • Плюсы: интерпретируемость, работает с нелинейностями.
  • Минусы: легко переобучается без ограничений.
Сетка гиперпараметров для гридсёрч
param_grid = {
    "criterion": ["gini", "entropy"],
    "max_depth": [None, 3, 5, 10, 20],
    "min_samples_split": [2, 5, 10],
    "min_samples_leaf": [1, 2, 4],
    "ccp_alpha": [0.0, 0.001, 0.01]
}

Logistic Regression (LR)

Принцип работы коротко

Линейная модель, которая через сигмоиду оценивает вероятность класса и порогом принимает решение.

Математическое обоснование подробно

p(y=1|x)=σ(w^Tx+b), где σ(z)=1/(1+e^-z). Параметры ищутся минимизацией log-loss: L=-Σ[ylogp+(1-y)log(1-p)] + λR(w).

Собственная реализация
class LogisticRegressionFromScratch:
    def fit(self, X, y):
        ...
    def predict(self, X):
        ...
Реализация sklearn
from sklearn.linear_model import LogisticRegression
model = LogisticRegression(max_iter=2000, solver="liblinear")
model.fit(X_train, y_train)
Описание гиперпараметров
  • C - обратная сила регуляризации.
  • penalty - тип регуляризации.
  • solver - оптимизатор.
  • class_weight - веса классов.
Интуиция использования (когда, какие данные, плюсы/минусы)
  • Когда: линейно разделимые или почти линейные данные, нужен быстрый и объяснимый baseline.
  • Плюсы: интерпретируемые коэффициенты, стабильная оптимизация.
  • Минусы: ограничена линейной границей.
Сетка гиперпараметров для гридсёрч
