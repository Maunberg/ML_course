1. Linear Regression

Базовая модель регрессии: предсказывает численное значение как линейную комбинацию признаков. Это первая модель, с которой обычно начинают регрессионный анализ.

Линейная модель Очень интерпретируема Нужна масштабная проверка допущений

Коротко: как работает

Модель пытается провести такую прямую, плоскость или гиперплоскость, чтобы предсказанные значения были как можно ближе к настоящим.

ŷ = Xw + b

Она подбирает веса w и смещение b, минимизируя ошибку между истинными значениями и прогнозами.

Подробное математическое объяснение

Линейная модель

ŷ = Xw + b

Функция потерь: MSE

L = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2

Градиенты

\frac{\partial L}{\partial w} = -\frac{2}{n}X^T(y - \hat{y}) \frac{\partial L}{\partial b} = -\frac{2}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)

Обновление параметров

w := w - \eta \cdot \frac{\partial L}{\partial w} b := b - \eta \cdot \frac{\partial L}{\partial b}

Смысл очень простой: если модель сильно ошибается, то градиент показывает, в какую сторону нужно изменить коэффициенты, чтобы уменьшить среднеквадратичную ошибку.

Нормальное уравнение

w = (X^T X)^{-1} X^T y

Это аналитическое решение линейной регрессии без градиентного спуска. Оно удобно на небольших данных, но плохо масштабируется на очень большие и может быть неустойчивым, если признаки сильно коррелируют.

Собственная реализация с объяснением

import numpy as np

class LinearRegressionFromScratch:
    def __init__(self, lr=0.01, n_iters=2000):
        self.lr = lr
        self.n_iters = n_iters
        self.w = None
        self.b = None

    def fit(self, X, y):
        X = np.asarray(X, dtype=float)
        y = np.asarray(y, dtype=float)

        n_samples, n_features = X.shape
        self.w = np.zeros(n_features)
        self.b = 0.0

        for _ in range(self.n_iters):
            y_pred = X @ self.w + self.b

            dw = (-2 / n_samples) * (X.T @ (y - y_pred))
            db = (-2 / n_samples) * np.sum(y - y_pred)

            self.w -= self.lr * dw
            self.b -= self.lr * db

        return self

    def predict(self, X):
        X = np.asarray(X, dtype=float)
        return X @ self.w + self.b

Здесь мы вручную считаем предсказания, ошибку, градиенты и обновляем параметры по градиентному спуску.

Плюсы, минусы и где применять

Плюсы

  • простая и быстрая;
  • отлично интерпретируется;
  • хорошо работает как baseline;
  • подходит для небольших и средних наборов данных.

Минусы

  • не ловит нелинейные зависимости;
  • чувствительна к выбросам;
  • страдает от мультиколлинеарности;
  • может недообучаться на сложных зависимостях.

Сколько данных и какие задачи

  • Минимум: десятки–сотни наблюдений.
  • Комфортно: сотни–десятки тысяч объектов.
  • Лучше всего: прогноз численных величин с примерно линейной зависимостью.

2. Ridge Regression

Это линейная регрессия с L2-регуляризацией. Она штрафует слишком большие коэффициенты, делая модель стабильнее и менее склонной к переобучению.

L2-регуляризация Устойчива к мультиколлинеарности Не зануляет признаки

Коротко: как работает

Ridge делает то же самое, что и обычная линейная регрессия, но дополнительно штрафует большие веса.

L = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2 + \alpha \sum_{j=1}^{d} w_j^2

Подробное математическое объяснение

Если признаки сильно коррелируют, линейная регрессия может давать нестабильные коэффициенты. Ridge добавляет L2-штраф:

R(w) = \sum_{j=1}^{d} w_j^2

Тогда функция потерь становится:

L = MSE + \alpha ||w||_2^2

Градиент:

\frac{\partial L}{\partial w} = -\frac{2}{n}X^T(y - \hat{y}) + 2\alpha w

Чем больше alpha, тем сильнее модель сжимает коэффициенты к нулю, но полностью обычно их не зануляет.

Собственная реализация

import numpy as np

class RidgeRegressionFromScratch:
    def __init__(self, lr=0.01, n_iters=2000, alpha=1.0):
        self.lr = lr
        self.n_iters = n_iters
        self.alpha = alpha
        self.w = None
        self.b = None

    def fit(self, X, y):
        X = np.asarray(X, dtype=float)
        y = np.asarray(y, dtype=float)
        n_samples, n_features = X.shape

        self.w = np.zeros(n_features)
        self.b = 0.0

        for _ in range(self.n_iters):
            y_pred = X @ self.w + self.b
            dw = (-2 / n_samples) * (X.T @ (y - y_pred)) + 2 * self.alpha * self.w
            db = (-2 / n_samples) * np.sum(y - y_pred)

            self.w -= self.lr * dw
            self.b -= self.lr * db

        return self

    def predict(self, X):
        X = np.asarray(X, dtype=float)
        return X @ self.w + self.b

Плюсы и минусы

  • устойчива при коррелированных признаках;
  • часто работает лучше обычной линейной регрессии;
  • стабильна на средних и больших данных.
  • не делает отбора признаков;
  • по-прежнему линейная;
  • нуждается в масштабировании признаков.

3. Lasso Regression

Линейная регрессия с L1-регуляризацией. В отличие от Ridge, Lasso может занулять коэффициенты и тем самым выполнять отбор признаков.

L1-регуляризация Отбор признаков Хороша при лишних признаках

Коротко: как работает

L = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2 + \alpha \sum_{j=1}^{d}|w_j|

Lasso старается не только минимизировать ошибку, но и делать модель более разреженной.

Подробное математическое объяснение

L1-регуляризация:

R(w) = \sum_{j=1}^{d}|w_j|

Особенность в том, что такой штраф геометрически способствует появлению нулевых коэффициентов.

Поэтому Lasso особенно полезна, когда признаков много и часть из них неинформативна.

Формально производная в нуле не определена, поэтому на практике используются специальные алгоритмы оптимизации, например coordinate descent.

Собственная реализация (упрощённая через субградиент)

import numpy as np

class LassoRegressionFromScratch:
    def __init__(self, lr=0.001, n_iters=3000, alpha=0.1):
        self.lr = lr
        self.n_iters = n_iters
        self.alpha = alpha
        self.w = None
        self.b = None

    def fit(self, X, y):
        X = np.asarray(X, dtype=float)
        y = np.asarray(y, dtype=float)
        n_samples, n_features = X.shape

        self.w = np.zeros(n_features)
        self.b = 0.0

        for _ in range(self.n_iters):
            y_pred = X @ self.w + self.b
            dw = (-2 / n_samples) * (X.T @ (y - y_pred)) + self.alpha * np.sign(self.w)
            db = (-2 / n_samples) * np.sum(y - y_pred)

            self.w -= self.lr * dw
            self.b -= self.lr * db

        return self

    def predict(self, X):
        X = np.asarray(X, dtype=float)
        return X @ self.w + self.b

Это учебная реализация. В реальных задачах sklearn-реализация обычно предпочтительнее из-за более устойчивой оптимизации.

  • иногда зануляет слишком агрессивно;
  • тоже требует масштабирования.

4. ElasticNet

ElasticNet объединяет Ridge и Lasso: одновременно уменьшает коэффициенты и может занулять часть из них. Часто это компромиссный вариант на сложных признаковых пространствах.

L1 + L2 Компромисс между Ridge и Lasso Полезна на широких датасетах

Коротко: как работает

L = MSE + \alpha \left( r \sum |w_j| + \frac{1-r}{2} \sum w_j^2 \right)

Где r = l1_ratio. Если r=1 — это почти Lasso, если r=0 — почти Ridge.

Подробное математическое объяснение

ElasticNet нужен, когда:

  • признаков много;
  • они коррелируют между собой;
  • хочется и регуляризацию, и частичный отбор признаков.
R(w) = r||w||_1 + \frac{1-r}{2}||w||_2^2

Это делает модель более устойчивой, чем чистый Lasso, но при этом сохраняет возможность зануления части коэффициентов.

Собственная реализация (учебная)

import numpy as np

class ElasticNetFromScratch:
    def __init__(self, lr=0.001, n_iters=4000, alpha=0.1, l1_ratio=0.5):
        self.lr = lr
        self.n_iters = n_iters
        self.alpha = alpha
        self.l1_ratio = l1_ratio
        self.w = None
        self.b = None

    def fit(self, X, y):
        X = np.asarray(X, dtype=float)
        y = np.asarray(y, dtype=float)
        n_samples, n_features = X.shape

        self.w = np.zeros(n_features)
        self.b = 0.0

        for _ in range(self.n_iters):
            y_pred = X @ self.w + self.b
            l1_grad = self.alpha * self.l1_ratio * np.sign(self.w)
            l2_grad = self.alpha * (1 - self.l1_ratio) * self.w
            dw = (-2 / n_samples) * (X.T @ (y - y_pred)) + l1_grad + l2_grad
            db = (-2 / n_samples) * np.sum(y - y_pred)

            self.w -= self.lr * dw
            self.b -= self.lr * db

        return self

    def predict(self, X):
        X = np.asarray(X, dtype=float)
        return X @ self.w + self.b

5. KNN Regressor

KNN-регрессия предсказывает значение по соседям: берёт ближайшие объекты и усредняет их целевые значения. Это простой, но мощный нелинейный метод.

Непараметрическая модель Ловит нелинейность Чувствительна к масштабу

Коротко: как работает

Для нового объекта ищутся k ближайших соседей, после чего прогноз строится как среднее или взвешенное среднее их значений.

\hat{y}(x)=\frac{1}{k}\sum_{i \in N_k(x)} y_i

Подробное математическое объяснение

Нужно выбрать метрику расстояния d(x, x_i), например евклидову:

d(x, x_i)=\sqrt{\sum_{j=1}^{d}(x_j-x_{ij})^2}

Далее выбираем множество ближайших соседей N_k(x). Если используем веса по расстоянию:

\hat{y}(x)=\frac{\sum_{i \in N_k(x)} w_i y_i}{\sum_{i \in N_k(x)} w_i}, \quad w_i = \frac{1}{d(x,x_i)+\varepsilon}

У KNN нет явного этапа обучения весов — модель фактически запоминает обучающую выборку.

Собственная реализация

import numpy as np

class KNNRegressorFromScratch:
    def __init__(self, n_neighbors=5, weights="uniform", p=2):
        self.n_neighbors = n_neighbors
        self.weights = weights
        self.p = p

    def fit(self, X, y):
        self.X_train = np.asarray(X, dtype=float)
        self.y_train = np.asarray(y, dtype=float)
        return self

    def _distance(self, x):
        return np.sum(np.abs(self.X_train - x) ** self.p, axis=1) ** (1 / self.p)

    def predict(self, X):
        X = np.asarray(X, dtype=float)
        preds = []

        for x in X:
            dists = self._distance(x)
            idx = np.argsort(dists)[:self.n_neighbors]
            neighbor_targets = self.y_train[idx]

            if self.weights == "distance":
                w = 1.0 / (dists[idx] + 1e-12)
                pred = np.sum(w * neighbor_targets) / np.sum(w)
            else:
                pred = np.mean(neighbor_targets)

            preds.append(pred)

        return np.array(preds)

Плюсы и минусы

  • умеет моделировать нелинейные зависимости;
  • прост в понимании;
  • хорош на локально гладких зависимостях.
  • плохо работает в высокой размерности;
  • медленный на больших датасетах;
  • очень чувствителен к масштабированию признаков.

6. Decision Tree Regressor

Дерево регрессии разбивает пространство признаков на области и в каждой области предсказывает, как правило, среднее значение целевой переменной.

Нелинейная модель Не требует масштабирования Легко переобучается

Коротко: как работает

Модель последовательно делит данные по порогам признаков так, чтобы уменьшать разброс целевой переменной в листьях дерева.

Подробное математическое объяснение

На каждом узле ищется лучшее разбиение по признаку и порогу. Критерий — уменьшение ошибки, чаще всего MSE:

I(S) = \frac{1}{|S|}\sum_{i \in S}(y_i - \bar{y}_S)^2

Качество сплита:

\Delta I = I(parent) - \frac{|S_L|}{|S|}I(S_L) - \frac{|S_R|}{|S|}I(S_R)

Выбирается то разбиение, которое сильнее всего уменьшает неоднородность.

Собственная реализация (упрощённая)

import numpy as np

class DecisionTreeRegressorNode:
    def __init__(self, feature=None, threshold=None, left=None, right=None, value=None):
        self.feature = feature
        self.threshold = threshold
        self.left = left
        self.right = right
        self.value = value

class DecisionTreeRegressorFromScratch:
    def __init__(self, max_depth=6, min_samples_split=2):
        self.max_depth = max_depth
        self.min_samples_split = min_samples_split

    def _mse(self, y):
        if len(y) == 0:
            return 0
        return np.mean((y - np.mean(y)) ** 2)

    def _best_split(self, X, y):
        best_gain = -1
        best_feature, best_threshold = None, None
        parent_error = self._mse(y)
        n_samples, n_features = X.shape

        for feature in range(n_features):
            thresholds = np.unique(X[:, feature])
            for threshold in thresholds:
                left_idx = X[:, feature] <= threshold
                right_idx = ~left_idx
                if left_idx.sum() == 0 or right_idx.sum() == 0:
                    continue

                left_error = self._mse(y[left_idx])
                right_error = self._mse(y[right_idx])
                weighted_error = (left_idx.sum() / n_samples) * left_error + (right_idx.sum() / n_samples) * right_error
                gain = parent_error - weighted_error

                if gain > best_gain:
                    best_gain = gain
                    best_feature = feature
                    best_threshold = threshold

        return best_feature, best_threshold

    def _build(self, X, y, depth):
        value = np.mean(y)
        if depth >= self.max_depth or len(y) < self.min_samples_split:
            return DecisionTreeRegressorNode(value=value)

        feature, threshold = self._best_split(X, y)
        if feature is None:
            return DecisionTreeRegressorNode(value=value)

        left_idx = X[:, feature] <= threshold
        right_idx = ~left_idx

        left = self._build(X[left_idx], y[left_idx], depth + 1)
        right = self._build(X[right_idx], y[right_idx], depth + 1)
        return DecisionTreeRegressorNode(feature, threshold, left, right, value=None)

    def fit(self, X, y):
        X = np.asarray(X, dtype=float)
        y = np.asarray(y, dtype=float)
        self.root = self._build(X, y, 0)
        return self

    def _predict_one(self, x, node):
        if node.value is not None:
            return node.value
        if x[node.feature] <= node.threshold:
            return self._predict_one(x, node.left)
        return self._predict_one(x, node.right)

    def predict(self, X):
        X = np.asarray(X, dtype=float)
        return np.array([self._predict_one(x, self.root) for x in X])

Реализация sklearn

from sklearn.tree import DecisionTreeRegressor

model = DecisionTreeRegressor(
    criterion="squared_error",
    max_depth=6,
    min_samples_split=5,
    min_samples_leaf=2,
    random_state=42
)
model.fit(X_train, y_train)
y_pred = model.predict(X_test)

  • criterion — функция качества сплита.

Плюсы, минусы и задачи

  • ловит сложные нелинейности;
  • не требует нормализации признаков;
  • работает и с числовыми, и с категориальными преобразованными признаками.
  • легко переобучается;
  • нестабильно к небольшим изменениям данных;
  • часто проигрывает ансамблям вроде Random Forest и Gradient Boosting.

Когда особенно полезно

  • Есть нелинейные зависимости и много пороговых эффектов.
  • Нужна интерпретируемость логики разбиений.
  • Нужен сильный baseline перед ансамблями деревьев.
