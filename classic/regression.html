Linear Regression

Принцип работы коротко

Модель аппроксимирует цель линейной комбинацией признаков: y_hat = Xw + b.

Математическое обоснование подробно

Оптимизируем MSE: L = (1/n) * sum((y_i - y_hat_i)^2). При полном ранге решение через нормальное уравнение: w = (X^T X)^(-1) X^T y. При больших данных обычно применяют градиентный спуск: dw = (2/n) X^T (Xw+b-y), db = (2/n) sum(Xw+b-y). Пример: если рост цели почти линейно зависит от одного признака, коэффициент w показывает вклад единичного изменения признака в прогноз.

Собственная реализация (полная, без sklearn)
import numpy as np

class LinearRegressionFromScratch:
    def __init__(self, lr=0.01, n_iter=5000):
        self.lr = lr
        self.n_iter = n_iter

    def fit(self, X, y):
        X = np.asarray(X, dtype=float)
        y = np.asarray(y, dtype=float)
        n, m = X.shape
        self.w = np.zeros(m)
        self.b = 0.0
        for _ in range(self.n_iter):
            y_hat = X @ self.w + self.b
            err = y_hat - y
            dw = (2.0 / n) * (X.T @ err)
            db = (2.0 / n) * np.sum(err)
            self.w -= self.lr * dw
            self.b -= self.lr * db
        return self

    def predict(self, X):
        X = np.asarray(X, dtype=float)
        return X @ self.w + self.b
Реализация sklearn
from sklearn.linear_model import LinearRegression

model = LinearRegression()
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
Описание гиперпараметров sklearn (подробно)
  • fit_intercept: добавлять ли свободный член b.
  • copy_X: копирование матрицы X для безопасности операций.
  • positive: ограничение коэффициентов неотрицательностью.
  • n_jobs: параллелизация при некоторых конфигурациях.
Интуиция использования

Лучший интерпретируемый baseline для численной цели. Хороша при почти линейной зависимости и как контрольная точка для более сложных моделей.

Ridge Regression

Принцип работы коротко

Линейная регрессия с L2-регуляризацией, уменьшающей величины коэффициентов.

Математическое обоснование подробно

L = ||y - Xw||^2 + alpha * ||w||^2. Регуляризация снижает дисперсию и помогает при мультиколлинеарности. Аналитически: w = (X^T X + alpha I)^(-1) X^T y. При alpha=0 получаем обычную линейную регрессию.

Собственная реализация (полная, без sklearn)
import numpy as np

class RidgeFromScratch:
    def __init__(self, alpha=1.0):
        self.alpha = alpha

    def fit(self, X, y):
        X = np.asarray(X, dtype=float)
        y = np.asarray(y, dtype=float)
        m = X.shape[1]
        I = np.eye(m)
        self.w = np.linalg.solve(X.T @ X + self.alpha * I, X.T @ y)
        self.b = 0.0
        return self

    def predict(self, X):
        X = np.asarray(X, dtype=float)
        return X @ self.w + self.b
Реализация sklearn
from sklearn.linear_model import Ridge

model = Ridge(alpha=1.0)
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
Описание гиперпараметров sklearn (подробно)
  • alpha: сила L2-регуляризации (чем больше, тем сильнее сжатие коэффициентов).
  • solver: метод решения (auto, svd, cholesky, sag, lbfgs).
  • fit_intercept: свободный член.
  • max_iter, tol: критерии сходимости для итерационных решателей.
Интуиция использования

Выбирай, когда признаки коррелируют и обычная линейная регрессия нестабильна.

Сетка гиперпараметров для GridSearchCV

    "alpha": [0.01, 0.1, 1.0, 10.0, 100.0],
    "solver": ["auto", "svd", "sag"],
    "fit_intercept": [True, False]
}

Lasso

Принцип работы коротко

Линейная регрессия с L1-регуляризацией, которая зануляет часть коэффициентов.

Математическое обоснование подробно

L = ||y-Xw||^2 + alpha * ||w||_1. L1 штраф создает разреженные решения и выполняет встроенный отбор признаков. Часто решается coordinate descent: обновляем по одному коэффициенту через soft-thresholding.

Собственная реализация (полная, без sklearn)
import numpy as np

class LassoFromScratch:
    def __init__(self, alpha=0.1, n_iter=3000):
        self.alpha = alpha
        self.n_iter = n_iter

    def fit(self, X, y):
        X = np.asarray(X, dtype=float)
        y = np.asarray(y, dtype=float)
        n, m = X.shape
        self.w = np.zeros(m)
        self.b = y.mean()
        y_c = y - self.b

        for _ in range(self.n_iter):
            for j in range(m):
                r = y_c - (X @ self.w) + X[:, j] * self.w[j]
                rho = X[:, j] @ r
                z = np.sum(X[:, j] ** 2)
                if rho < -self.alpha / 2:
                    self.w[j] = (rho + self.alpha / 2) / z
                elif rho > self.alpha / 2:
                    self.w[j] = (rho - self.alpha / 2) / z
                else:
                    self.w[j] = 0.0
        return self

    def predict(self, X):
        X = np.asarray(X, dtype=float)
        return X @ self.w + self.b
Реализация sklearn
from sklearn.linear_model import Lasso

model = Lasso(alpha=0.01, max_iter=5000)
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
Описание гиперпараметров sklearn (подробно)
  • alpha: сила L1-регуляризации.
  • max_iter, tol: сходимость coordinate descent.
  • selection: порядок обновления координат (cyclic/random).
  • warm_start: использовать предыдущее решение для ускорения.
Интуиция использования

Подходит, когда признаков много и нужна интерпретируемая модель с отбором важнейших факторов.

ElasticNet

Принцип работы коротко

Сочетает L1 и L2-регуляризацию, балансируя отбор признаков и устойчивость.

Математическое обоснование подробно

L = ||y-Xw||^2 + alpha * (l1_ratio*||w||_1 + (1-l1_ratio)*||w||^2). При l1_ratio=1 это Lasso, при 0 - Ridge-подобный режим.

Собственная реализация (полная, без sklearn)
import numpy as np

class ElasticNetFromScratch:
    def __init__(self, alpha=0.1, l1_ratio=0.5, lr=0.01, n_iter=4000):
        self.alpha = alpha
        self.l1_ratio = l1_ratio
        self.lr = lr
        self.n_iter = n_iter

    def fit(self, X, y):
        X = np.asarray(X, dtype=float)
        y = np.asarray(y, dtype=float)
        n, m = X.shape
        self.w = np.zeros(m)
        self.b = 0.0
        for _ in range(self.n_iter):
            y_hat = X @ self.w + self.b
            err = y_hat - y
            grad_l2 = (1 - self.l1_ratio) * self.w
            grad = (2 / n) * (X.T @ err) + self.alpha * grad_l2
            self.w -= self.lr * grad
            self.w = np.sign(self.w) * np.maximum(0, np.abs(self.w) - self.lr * self.alpha * self.l1_ratio)
            self.b -= self.lr * (2 / n) * np.sum(err)
        return self

    def predict(self, X):
        X = np.asarray(X, dtype=float)
        return X @ self.w + self.b
Реализация sklearn
from sklearn.linear_model import ElasticNet

model = ElasticNet(alpha=0.01, l1_ratio=0.7, max_iter=5000)
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
Описание гиперпараметров sklearn (подробно)
  • alpha: общая сила регуляризации.
  • l1_ratio: доля L1 в регуляризации.
  • max_iter, tol, selection, warm_start.
Интуиция использования

Полезен при группах коррелирующих признаков: мягче Lasso, но все еще способен занулять веса.

Сетка гиперпараметров для GridSearchCV

    "alpha": [1e-4, 1e-3, 1e-2, 1e-1],
    "l1_ratio": [0.1, 0.3, 0.5, 0.7, 0.9],
    "max_iter": [3000, 5000, 8000]
}

RandomForestRegressor

Принцип работы коротко

Ансамбль деревьев регрессии на бутстрэп-выборках, финальный прогноз - среднее прогнозов деревьев.

Математическое обоснование подробно

Каждое дерево минимизирует внутрилетсевую дисперсию сплитов, ансамблирование снижает variance. Идея: E[mean(tree)] сохраняет смещение, но сильно уменьшает дисперсию при слабой корреляции деревьев.

Собственная реализация (полная, без sklearn)
import numpy as np

class DecisionStumpReg:
    def fit(self, X, y):
        best_f, best_t, best_loss = None, None, np.inf
        n, m = X.shape
        for f in range(m):
            for t in np.unique(X[:, f]):
                l = X[:, f] <= t
                r = ~l
                if l.sum() == 0 or r.sum() == 0:
                    continue
                yl, yr = y[l], y[r]
                loss = ((yl - yl.mean()) ** 2).sum() + ((yr - yr.mean()) ** 2).sum()
                if loss < best_loss:
                    best_loss, best_f, best_t = loss, f, t
        self.f, self.t = best_f, best_t
        l = X[:, self.f] <= self.t
        r = ~l
        self.left_val = y[l].mean()
        self.right_val = y[r].mean()

    def predict(self, X):
        return np.where(X[:, self.f] <= self.t, self.left_val, self.right_val)

class RandomForestRegressorFromScratch:
    def __init__(self, n_estimators=100, random_state=42):
        self.n_estimators = n_estimators
        self.rng = np.random.default_rng(random_state)

    def fit(self, X, y):
        X = np.asarray(X, dtype=float)
        y = np.asarray(y, dtype=float)
        n = len(X)
        self.trees = []
        for _ in range(self.n_estimators):
            idx = self.rng.integers(0, n, n)
            tree = DecisionStumpReg()
            tree.fit(X[idx], y[idx])
            self.trees.append(tree)
        return self

    def predict(self, X):
        X = np.asarray(X, dtype=float)
        preds = np.array([t.predict(X) for t in self.trees])
        return preds.mean(axis=0)
Реализация sklearn
from sklearn.ensemble import RandomForestRegressor

model = RandomForestRegressor(n_estimators=300, random_state=42)
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
Описание гиперпараметров sklearn (подробно)
  • n_estimators: количество деревьев.
  • max_depth, min_samples_split, min_samples_leaf: контроль сложности дерева.
  • max_features: сколько признаков рассматривать при сплите.
  • bootstrap, oob_score: бутстрэп и OOB-валидация.
  • criterion: функция качества сплита (squared_error, absolute_error).
Интуиция использования

Отличный выбор для нелинейной регрессии на табличных данных с минимальной инженерией признаков.
