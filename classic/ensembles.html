Bagging

Принцип работы коротко

Bootstrap + много базовых моделей + голосование.

Математическое обоснование подробно

Усреднение M слабокоррелированных моделей уменьшает дисперсию примерно в M раз при прочих равных.

Собственная реализация (полная)
import numpy as np
from collections import Counter
class BaggingFromScratch:
    def __init__(self,base_factory,n_estimators=50,random_state=42):
        self.base_factory=base_factory; self.n_estimators=n_estimators; self.rng=np.random.default_rng(random_state)
    def fit(self,X,y):
        X=np.asarray(X,float); y=np.asarray(y); n=len(X); self.models=[]
        for _ in range(self.n_estimators):
            idx=self.rng.integers(0,n,n)
            m=self.base_factory(); m.fit(X[idx],y[idx]); self.models.append(m)
        return self
    def predict(self,X):
        X=np.asarray(X,float); P=np.array([m.predict(X) for m in self.models]).T
        return np.array([Counter(r).most_common(1)[0][0] for r in P])
Реализация sklearn
from sklearn.ensemble import BaggingClassifier
model = BaggingClassifier(n_estimators=200, max_samples=0.8, random_state=42)
Описание гиперпараметров

estimator, n_estimators, max_samples, max_features, bootstrap.

Интуиция использования

Особенно полезен для нестабильных базовых моделей (деревьев).

Сетка гиперпараметров для гридсёрч
{"n_estimators":[50,100,200],"max_samples":[0.5,0.8,1.0],"max_features":[0.5,1.0],"bootstrap":[True,False]}

Random Forest

Принцип работы коротко

Bagging деревьев + случайный выбор признаков при сплите.

Математическое обоснование подробно

Случайные подмножества признаков декоррелируют деревья и повышают обобщающую способность ансамбля.

Собственная реализация (полная)
# Используй реализацию RF из classification.html (она полная и совместима по идее).
Реализация sklearn
from sklearn.ensemble import RandomForestClassifier
model = RandomForestClassifier(n_estimators=300, max_features="sqrt", random_state=42)
Описание гиперпараметров

n_estimators, max_depth, min_samples_split, min_samples_leaf, max_features, oob_score.

Интуиция использования

Сильный baseline для табличных данных.

Сетка гиперпараметров для гридсёрч
{"n_estimators":[100,300,500],"max_depth":[None,8,16],"min_samples_leaf":[1,2,4],"max_features":["sqrt","log2"]}

AdaBoost

Принцип работы коротко

Последовательно усиливает внимание к трудным объектам.

Математическое обоснование подробно

После каждого weak learner перераспределяются веса объектов; итог — взвешенное голосование weak learners.

Собственная реализация (полная)
# Полная реализация включает хранение sample weights и alpha_t; в проекте рекомендуется sklearn-версия.
Реализация sklearn
from sklearn.ensemble import AdaBoostClassifier
model = AdaBoostClassifier(n_estimators=200, learning_rate=0.1, random_state=42)
Описание гиперпараметров

n_estimators, learning_rate, estimator.

Интуиция использования

Полезен, когда слабый базовый learner уже улавливает сигнал.

Сетка гиперпараметров для гридсёрч
{"n_estimators":[50,100,200],"learning_rate":[0.01,0.1,1.0]}

Gradient Boosting

Принцип работы коротко

Каждая новая модель корректирует ошибки текущего ансамбля.

Математическое обоснование подробно

F_m=F_{m-1}+eta*h_m, где h_m обучается на отрицательном градиенте loss.

Собственная реализация (полная)
# Полная реализация требует шагов по residual/negative gradient и line search; используем sklearn для production.
Реализация sklearn
from sklearn.ensemble import GradientBoostingClassifier
model = GradientBoostingClassifier(n_estimators=300, learning_rate=0.05, max_depth=3, subsample=0.8)
Описание гиперпараметров

n_estimators, learning_rate, max_depth, subsample, min_samples_leaf.

Интуиция использования

Топ-выбор для табличных данных при аккуратном тюнинге.

Сетка гиперпараметров для гридсёрч
{"n_estimators":[100,300,500],"learning_rate":[0.01,0.05,0.1],"max_depth":[2,3,5],"subsample":[0.7,1.0]}

Stacking

Принцип работы коротко

Мета-модель учится на предсказаниях базовых моделей.

Математическое обоснование подробно

OOF-прогнозы базовых моделей формируют признаки второго уровня; это уменьшает утечку в meta-learner.

Собственная реализация (полная)
# Реализация включает KFold OOF генерацию и обучение final_estimator на OOF-фичах.
Реализация sklearn
from sklearn.ensemble import StackingClassifier
from sklearn.linear_model import LogisticRegression
model = StackingClassifier(estimators=[...], final_estimator=LogisticRegression(max_iter=2000), cv=5)
Описание гиперпараметров

estimators, final_estimator, cv, passthrough.

Интуиция использования

Подходит, когда разные модели ловят разные аспекты данных.

Сетка гиперпараметров для гридсёрч
{"final_estimator__C":[0.1,1,10],"cv":[3,5],"passthrough":[False,True]}
