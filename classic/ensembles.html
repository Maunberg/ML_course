Bagging

Принцип работы коротко

Много моделей на бутстрэпе + голосование/усреднение.

Математическое обоснование подробно

Усреднение снижает дисперсию предсказаний.

Собственная реализация
# bootstrap -> fit base learners -> aggregate
Реализация sklearn
BaggingClassifier(n_estimators=200).fit(X_train,y_train)
Описание гиперпараметров

n_estimators, max_samples, max_features.

Интуиция использования

Стабилизирует нестабильные базовые модели.

Сетка гиперпараметров для гридсёрч
{"n_estimators":[50,100,200],"max_samples":[0.5,0.8,1.0]}

AdaBoost

Принцип работы коротко

Фокусируется на трудно классифицируемых примерах.

Математическое обоснование подробно

Экспоненциальная потеря и ре-взвешивание объектов.

Собственная реализация
# sample weights update after each weak learner
Реализация sklearn
AdaBoostClassifier(n_estimators=200,learning_rate=0.1).fit(X_train,y_train)
Описание гиперпараметров

n_estimators, learning_rate.

Интуиция использования

Сильна при чистом сигнале и простых weak learners.

Сетка гиперпараметров для гридсёрч
{"n_estimators":[50,100,200],"learning_rate":[0.01,0.1,1.0]}

Gradient Boosting

Принцип работы коротко

Последовательно исправляет остаточные ошибки ансамбля.

Математическое обоснование подробно

Функциональный градиентный спуск в пространстве моделей.

Собственная реализация
# fit tree on residual/negative gradient
Реализация sklearn
GradientBoostingClassifier().fit(X_train,y_train)
Описание гиперпараметров

n_estimators, learning_rate, max_depth, subsample.

Интуиция использования

Очень сильный baseline на табличных данных.

Сетка гиперпараметров для гридсёрч
{"n_estimators":[100,300,500],"learning_rate":[0.01,0.05,0.1],"max_depth":[2,3,5]}

Extra Trees

Принцип работы коротко

Как RF, но со случайными порогами сплитов.

Математическое обоснование подробно

Больше рандомизации -> меньше дисперсия, иногда чуть выше смещение.

Собственная реализация
# random thresholds per feature when splitting
Реализация sklearn
ExtraTreesClassifier(n_estimators=400).fit(X_train,y_train)
Описание гиперпараметров

n_estimators, max_depth, max_features.

Интуиция использования

Быстрее и устойчиво на шумных табличных данных.

Сетка гиперпараметров для гридсёрч
{"n_estimators":[200,400],"max_depth":[None,10,20],"max_features":["sqrt","log2"]}

Stacking

Принцип работы коротко

Комбинирует предсказания разных моделей через мета-модель.

Математическое обоснование подробно

Мета-уровень учится на out-of-fold прогнозах базовых моделей.

Собственная реализация
# K-fold OOF predictions -> fit meta model
Реализация sklearn
StackingClassifier(estimators=[...],final_estimator=LogisticRegression()).fit(X_train,y_train)
Описание гиперпараметров

Список базовых моделей, мета-модель, cv.

Интуиция использования

Когда разные модели ловят разные паттерны.

Сетка гиперпараметров для гридсёрч
{"final_estimator__C":[0.1,1,10],"cv":[3,5]}
